<h3 align="center">CTO · Head of Engineering · Building production AI, agentic systems, and secure data infrastructure</h3>

---

I build and lead engineering for AI products that have to hold up in regulated, high-stakes environments. My focus is the boring-but-hard part: making LLM systems reliable, secure, cost-efficient, and compliant at scale.

- **CTO @ Golden Gate Innovations (GGI)** — Silicon Valley venture studio. I set technical direction across portfolio companies spanning **PropTech, Food Tech, Crypto/Blockchain, and Deep Tech**.
- **Head of Engineering & SOC Head** — A legal AI SaaS processing sensitive case data for law firms, under **SOC 2 Type II** and **HIPAA** compliance.
- Background spanning application security, distributed data systems, and domain-driven backend architecture.

### What I work on

- **Agentic systems** — multi-agent orchestration, tool-use pipelines, model harnesses, and production coding-agent workflows.
- **LLM infrastructure** — multi-provider routing (Anthropic, OpenAI, Azure AI Foundry, AWS Bedrock, OpenRouter), prompt caching, cost analysis, and private/isolated model endpoints for enterprise data isolation.
- **Document & ingestion pipelines** — OCR provider routing, duplicate detection, and DDD-refactored ingestion services on a Kafka / Iceberg / MinIO / Spark medallion-architecture data foundry.
- **Security & compliance** — vendor security assessments, SOC incident-response design, WAF tuning, TLS diagnostics, pen testing (IDOR, auth bypass), and cryptography.
- **Systems architecture** — Node.js / TypeScript with DDD patterns (static factories, repository/service separation, Drizzle ORM), plus distributed data systems (Spark, Flink, ClickHouse, Kafka).

### Interests

GPU programming (CUDA, Triton kernels), defense technology, and a research-grounded, critical perspective on what LLMs can and cannot actually do.
